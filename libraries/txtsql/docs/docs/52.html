SQL Syntax

These sections help one to better understand the SQL capabilities of txtSQL. txtSQL introduced SQL syntax as part of their official core release as of version 3.0.0 Beta.

txtSQL does NOT support all SQL syntax; just the syntax in the documented sections are supported.

To be able to read the syntax notation, please make note of the following:

  • The character "|" (pipeline) is used to show that there is an option between what's on the left of it and the right. For example, "db.table|table" means either specify "db.table" or "table"
  • Anything between the square brackets ([...]) is optional

    Please note that anything not inside of quotation marks is not interpreted as a string, so please enclose all string literals inside of either single or double quotes
